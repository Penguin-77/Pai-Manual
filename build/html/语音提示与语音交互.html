8.5. 语音提示与语音交互

本章介绍机器人的语音功能，涵盖语音提示与语音交互两部分。

语音提示是机器人在特定事件发生时主动向用户播报信息的方式，帮助用户及时掌握设备状态；语音交互则允许用户直接以语音向机器人下达指令或进行对话问答，实现更自然、便捷的操作体验。

两者共同构成了人机之间完整的语音沟通机制。

8.5.1. 语音提示

8.5.1.1. 关于语音提示

机器人内置语音助手 「派仔」（男童声）。开机自检阶段由启动程序直接播放；进入正常运行后，由控制器与业务接口在关键状态变化时触发播放。

当前的语音分类有：

表 8.7 语音分类

类别

大致时段

目的

开机自检

按下电源、系统启动后的约 1 分钟内

告知编号、自检进度、通讯是否正常

待安装升级

开机时若检测到升级包

备注

在下列情况中，机器人 不会 进行播报，或可能无声，均属正常现象：

  • 现场关闭了启动语音（运维配置 JKKHAN_STARTUP_VOICE=0）

  • 喇叭音量被调低，或外接音响未接好

  • 两条提示间隔过短，后一条被系统自动合并/跳过（约 3 秒内）

  • 控制源实际未变化（例如重复登录 App）——不会重复播报「切换到应用」

若完全无声，请检查机器人外放音量，或联系售后确认语音功能是否启用。

8.5.1.2. 开机流程与语音时间线

从按下电源到可操控，典型顺序如下（部分步骤视现场配置可能省略）：

[上电]
   ↓
① 身份与自检开始 ——「我是派仔 X 号，开始上电自检」
   ↓
（可选）电源模块检测失败 ——「未检测到电源管理模块，执行上电失败」
   ↓
（可选）发现待安装升级包 ——「检测到 XX 升级包」→「开始升级…」→ 成功/失败
   ↓
（可选）SCB 类升级成功 ——「请断电重启」（连续 3 次，间隔约 3 秒）
   ↓
② 通讯自检结果 ——「自检通过」 或 「自检超时，仅检测到 N 个从站，期望 30 个从站」
   ↓
[系统进入正常运行]
   ↓
③ 全部关节通讯就绪 ——「通讯正常，所有关节已就绪」
   ↓
（用户/App 操作上使能后）
④ 使能结果 —— 成功或失败（见第 4 节）
   ↓
（日常使用）模式切换、控制源切换、故障、校零等

备注

机器人编号：开机时播报「我是派仔 X 号」中的 X，为产线写入的 SN1 生产流水号（5 位，如 SN1=2012345 则播报「12345 号」）。未写入或读取失败时，回退为默认 1 号。

8.5.1.5. 常见问题（FAQ）

Q1：开机只听到「我是派仔 1 号…」，但我这台不是 1 号 A：开机编号来自 SN1 生产流水号。若听到 1 号，多为 SN1 未写入、读取失败，或流水号确为 1。产线应在交付前写入正确 SN1；也可由售后核对 SN 页读回值。

Q2：开机只听到「我是派仔 X 号…」，之后很久没声音？ A：可能仍在等待 30 个模块上线（最长约 40 秒），或正在执行待安装升级（可能数分钟）。若超过 5 分钟仍无「自检通过/超时」，请联系售后。

Q3：「自检超时」后机器人还能演示吗？ A：不建议正式演示。缺失模块对应关节可能无力、乱报故障或运动不协调。应优先修复通讯。

Q4：为什么使能时没有语音？ A：可能语音间隔内已有其他播报被跳过；或使能未真正成功（查 App 状态）；或语音功能被关闭。

Q5：故障语音能告诉是哪个具体关节吗？ A：语音按身体部位（如「左腿」）播报，并给出错误类型中文说明，不会念英文关节名。更细信息需售后通过日志查看。

Q6：同一次故障会反复播报吗？ A：同一次故障持续期间通常只播报一次；故障消除后会播报「已排除」；再次故障会再次播报。

Q7：如何关闭语音？ A：一般用户无需关闭。若现场需静音（如展会），请联系运维人员配置，不建议自行改系统。

Q8：App 显示切换成功但没听到语音？ A：切换可能实际成功，语音因间隔限制或音量未播出。以 App 状态与机器人动作为准；若两者不一致再联系支持。

8.5.2. 语音交互

8.5.2.1. 开启语音交互

在项目根目录执行：

python main.py --stt --tts

其中：

  • --stt：开启实时语音识别；

  • --tts：开启语音合成和播报；

  • 运行前需要设置 DASHSCOPE_API_KEY；

  • 默认使用 DashScope STT，默认语音为 dashscope 链路。

Windows 示例：

$env:DASHSCOPE_API_KEY = "sk-xxxxxxxx"
python main.py --stt --tts

8.5.2.2. 关闭语音交互

  • 关闭语音输入：不添加 --stt，例如 python main.py，使用键盘输入。

  • 关闭语音输出：不添加 --tts，程序只输出文字。

  • 停止当前程序：在终端按 Ctrl+C。

  • 关闭当前语音会话但不退出程序：说“再见”“拜拜”“休息吧”“结束对话”“退出”“结束”或“关闭”。程序会回到等待唤醒状态。

如需调整休眠、唤醒词、音频保存等行为，请修改 config.py 中对应的 AppConfig 配置；修改后需要重启程序。

8.5.2.7. 常见问题与故障排查

使用中遇到问题时，请先查阅以下常见问题和解决方案。

Q1：我说了唤醒词，派仔没反应怎么办？

A：请依次检查： * 确认麦克风已正确连接且未被静音

  • 检查环境噪音是否过大（尝试走近到 1m 左右再说）

  • API Key 已正确设置；

  • 程序启动时使用了 --stt --tts 参数；

Q2：我说了指令，但派仔理解成了别的东西？

A：语音识别在嘈杂环境下可能出现同音字错误（如将"挥手"识别为"回手"）。系统有内置的 ASR 纠错机制，大多数情况下能自动修正。如果持续识别不准，请检查环境噪音是否超标，或尝试换一种表达方式。

Q3：我说了手势指令（如「挥挥手」），派仔回答但机器人没有动？

A：

  • 动作指令未被正确识别；

  • 机器人当前状态不允许执行该动作；

  • 软件未连接真机；

  • 动作被安全策略拦截。

Q4：派仔说话时我能打断吗？

A：当前版本默认不支持语音打断，派仔在说话期间麦克风处于静音状态。但如果使用科大讯飞的麦克风，支持打断。.

Q5：如何确认是否保存了录用

A：请检查 AUDIO_SAVE_ENABLED 的配置值：

未设置，或设置为 0、false 时，默认不持续保存录音；

设置为 1 或 true 后，录音将写入 audio/ 或指定目录。
