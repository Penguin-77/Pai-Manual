语音提示与语音交互
###########################

本章介绍机器人的语音功能，涵盖语音提示与语音交互两部分。

语音提示是机器人在特定事件发生时主动向用户播报信息的方式，帮助用户及时掌握设备状态；语音交互则允许用户直接以语音向机器人下达指令或进行对话问答，实现更自然、便捷的操作体验。

两者共同构成了人机之间完整的语音沟通机制。

语音提示
**********

关于语音提示
================

机器人内置语音助手 **「派仔」**\ （男童声）。开机自检阶段由启动程序直接播放；进入正常运行后，由控制器与业务接口在关键状态变化时触发播放。

当前的语音分类有：

.. list-table:: 语音分类
   :widths: 50 50 90
   :header-rows: 1

   * - 类别
     - 大致时段
     - 目的
   * - **开机自检**
     - 按下电源、系统启动后的约 1 分钟内
     - 告知编号、自检进度、通讯是否正常
   * - **待安装升级**
     - 开机时若检测到升级包
     - 告知正在升级哪类固件、是否成功、是否需要断电
   * - **运行状态**
     - 机器人完成启动之后
     - 使能/通讯/故障/校零等安全与操作反馈
   * - **模式与控制源**
     - 使用 App 或切换控制方式时
     - 告知当前由谁控制、动作模式是否切换成功

.. note::

   在下列情况中，机器人 **不会** 进行播报，或可能无声，均属正常现象：

*   现场关闭了启动语音（运维配置 ``JKKHAN_STARTUP_VOICE=0``\ ）

*   喇叭音量被调低，或外接音响未接好

*   两条提示间隔过短，后一条被系统自动合并/跳过（约 3 秒内）

*   控制源实际未变化（例如重复登录 App）——不会重复播报「切换到应用」

**若完全无声**，请检查机器人外放音量，或联系售后确认语音功能是否启用。

开机流程与语音时间线
========================

从按下电源到可操控，典型顺序如下（部分步骤视现场配置可能省略）：

.. code-block:: text

   [上电]
      ↓
   ① 身份与自检开始 ——「我是派仔 X 号，开始上电自检」
      ↓
   （可选）电源模块检测失败 ——「未检测到电源管理模块，执行上电失败」
      ↓
   （可选）发现待安装升级包 ——「检测到 XX 升级包」→「开始升级…」→ 成功/失败
      ↓
   （可选）SCB 类升级成功 ——「请断电重启」（连续 3 次，间隔约 3 秒）
      ↓
   ② 通讯自检结果 ——「自检通过」 或 「自检超时，仅检测到 N 个从站，期望 30 个从站」
      ↓
   [系统进入正常运行]
      ↓
   ③ 全部关节通讯就绪 ——「通讯正常，所有关节已就绪」
      ↓
   （用户/App 操作上使能后）
   ④ 使能结果 —— 成功或失败（见第 4 节）
      ↓
   （日常使用）模式切换、控制源切换、故障、校零等

.. note::

   **机器人编号**：开机时播报「我是派仔 X 号」中的 X，为产线写入的 SN1 生产流水号（5 位，如 SN1=2012345 则播报「12345 号」）。未写入或读取失败时，回退为默认 1 号。

-------------------

.. list-table::
   :widths: 50 50
   :header-rows: 1

通讯自检 — 超时（警告）
-----------------------

.. list-table::
   :widths: 50 50
   :header-rows: 1

   * - 项目
     - 说明
   * - **何时播放**
     - 等待超时后，上线的从站数仍少于期望的 30 个
   * - **播报内容**
     - **「自检超时，仅检测到 N 个从站，期望 30 个从站」**\ （N 为实际检测数量）
   * - **含义**
     - 部分关节或模块未成功接入通讯网络
   * - **可能后果**
     - 系统\ **仍会继续启动**\ ，但对应关节可能无法使能、无反馈或频繁报警
   * - **建议排查**
     - ① 记录 N 与 30 的差值，判断大致缺失模块数量 ② 检查近期是否拆装过某条腿/臂 ③ 查看是否有关节 LED 或驱动器异常 ④ 重启一次；若 N 每次相同，重点查对应部位接线 ⑤ 联系售后并提供 N 的数值

正常运行阶段语音（启动完成后）
+++++++++++++++++++++++++++++++

以下语音在机器人主程序运行期间播放，与日常操作和安全密切相关。

全部关节通讯就绪
------------------

.. list-table::
   :widths: 50 50
   :header-rows: 1

   * - 项目
     - 说明
   * - **何时播放**
     - 所有关节驱动器进入正常通讯状态，且稳定约 1 秒后（仅首次）
   * - **播报内容**
     - **「通讯正常，所有关节已就绪」**
   * - **含义**
     - 硬件链路 OK，可以进行使能等操作
   * - **若未听到**
     - 可能启动阶段已报自检超时；或语音功能关闭；通讯可能仍在恢复中

部分关节通讯异常
-------------------

.. list-table::
   :widths: 50 50
   :header-rows: 1

   * - 项目
     - 说明
   * - **何时播放**
     - 原先全部就绪，之后有任一关节退出正常通讯状态
   * - **播报内容**
     - **「注意，部分关节通讯未就绪，请检查接线与上电」**
   * - **含义**
     - 通讯中断或不稳定
   * - **可能后果**
     - 若当时处于使能状态，系统可能\ **自动下使能**\ 以保护安全
   * - **建议排查**
     - ① 立即停止使用 ② 检查报错部位线缆、接插件 ③ 重启；若反复出现，记录发生时的动作（如某条腿大幅摆动后）并联系售后

关节报警（故障）
-------------------

.. list-table::
   :widths: 50 50
   :header-rows: 1

   * - 项目
     - 说明
   * - **何时播放**
     - 任一关节进入故障/报警状态，且稳定约半秒后
   * - **播报内容（详细模式，默认）**
     - **「注意，[部位]，[错误原因]，请立即下使能并检查」**
   * - **播报内容（简化模式）**
     - **「关节报警，请立即下使能并检查」**
   * - **多部位同时报警**
     - **「注意，多个部位关节报警，请立即下使能并检查」**

**部位包括**\ ：头部、腰部、左臂、右臂、左腿、右腿。

**常见错误原因举例**\ （完整列表见附录 A）：

.. list-table::
   :widths: 50 50 50
   :header-rows: 1

   * - 您可能听到的
     - 大致含义
     - 现场可尝试
   * - 伺服过压 / 伺服欠压
     - 供电异常
     - 检查电源、重启
   * - 电机过温 / 伺服过温
     - 过热保护
     - 停机冷却 10～15 分钟
   * - 电机过载
     - 负载过大或卡阻
     - 下使能，检查是否碰撞、缠绕
   * - 编码器校零失败
     - 零点数据异常
     - 勿继续使能；联系售后校零
   * - EtherCAT 通讯故障
     - 通讯中断
     - 查线缆、重启
   * - 位置偏差过大
     - 跟踪误差超限
     - 下使能，检查是否被阻挡
   * - 上使能失败
     - 驱动器内部拒绝使能
     - 下使能后重启；仍失败联系售后
   * - 关节报警（原因未知）
     - 错误码未收录或无法识别
     - 记录发生时的动作，联系售后

**您必须做什么**\ ：

1.  **立即下使能**\ （App 或物理急停）

2.  不要继续发运动指令

3.  观察是否有冒烟、异响、异味

4.  故障消除并稳定约半秒后，会听到 **「关节故障已排除，可重新上使能」**

5.  确认原因前不要盲目再次使能

关节校零（校零）
--------------------

通过 App 或维护工具对单个或全部关节执行「校零」时：

.. list-table::
   :widths: 50 50 50
   :header-rows: 1

   * - 播报内容
     - 何时
     - 建议
   * - **「校零成功」**
     - 校零完成且位置验证通过
     - 可继续后续操作
   * - **「校零失败，请检查关节状态」**
     - 校零指令失败或位置未到位
     - 确认关节未使能、无故障、无机械卡阻后重试
   * - **「关节名称不存在」**
     - 指定了不存在的关节名
     - 检查 App 或工具中的关节名称是否正确

.. note::

   **安全提示**\ ：校零应在机器人\ **静止、下使能**\ 且\ **无人员触碰关节**\ 时进行；运动过程中不可校零。


电池电量过低
++++++++++++++++++++

.. list-table::
   :widths: 50 50
   :header-rows: 1

   * - 项目
     - 说明
   * - **「何时播放」**
     - 主电池相对电量（SOC）首次降至 20%、10%、5% 以下各一次；电量回升超过对应阈值约 2% 后再次下降，会重新提示
   * - **「播报内容（20%）」**
     - **电池电量低于百分之二十，请及时充电**
   * - **「播报内容（10%）」**
     - **电池电量低于百分之十，请及时充电**
   * - **「播报内容（5%）」**
     - **电池电量严重，请立即充电**
   * - 含义
     - 电池余量不足，继续作业可能导致意外下电
   * - 可能后果
     - 不会自动下使能，但电量继续下降可能导致系统断电
   * - 您需要做什么
     - ① 尽快连接充电器或更换满电电池 ② 低电量时避免大幅度、长时间运动 ③ 5% 提示后建议停止作业并充电

常见问题（FAQ）
==================

**Q1：开机只听到「我是派仔 1 号…」，但我这台不是 1 号**   A：开机编号来自 SN1 生产流水号。若听到 1 号，多为 SN1 未写入、读取失败，或流水号确为 1。产线应在交付前写入正确 SN1；也可由售后核对 SN 页读回值。

**Q2：开机只听到「我是派仔 X 号…」，之后很久没声音？**  A：可能仍在等待 30 个模块上线（最长约 40 秒），或正在执行待安装升级（可能数分钟）。若超过 5 分钟仍无「自检通过/超时」，请联系售后。

**Q3：「自检超时」后机器人还能演示吗？**   A：不建议正式演示。缺失模块对应关节可能无力、乱报故障或运动不协调。应优先修复通讯。

**Q4：为什么使能时没有语音？**   A：可能语音间隔内已有其他播报被跳过；或使能未真正成功（查 App 状态）；或语音功能被关闭。

**Q5：故障语音能告诉是哪个具体关节吗？**   A：语音按\ **身体部位**\ （如「左腿」）播报，并给出\ **错误类型**\ 中文说明，不会念英文关节名。更细信息需售后通过日志查看。

**Q6：同一次故障会反复播报吗？**   A：同一次故障持续期间通常\ **只播报一次**\ ；故障消除后会播报「已排除」；再次故障会再次播报。

**Q7：如何关闭语音？**   A：一般用户无需关闭。若现场需静音（如展会），请联系运维人员配置，不建议自行改系统。

**Q8：App 显示切换成功但没听到语音？**   A：切换可能实际成功，语音因间隔限制或音量未播出。以 App 状态与机器人动作为准；若两者不一致再联系支持。

语音交互
***********

开启语音交互
====================

在项目根目录执行：

.. code-block:: bash

    python main.py --stt --tts

其中：

- ``--stt``：开启实时语音识别；
- ``--tts``：开启语音合成和播报；
- 运行前需要设置 ``DASHSCOPE_API_KEY``；
- 默认使用 DashScope STT，默认语音为 ``dashscope`` 链路。

Windows 示例：

.. code-block:: powershell

    $env:DASHSCOPE_API_KEY = "sk-xxxxxxxx"
    python main.py --stt --tts

关闭语音交互
====================

- 关闭语音输入：不添加 ``--stt``，例如 ``python main.py``，使用键盘输入。
- 关闭语音输出：不添加 ``--tts``，程序只输出文字。
- 停止当前程序：在终端按 `Ctrl+C`。
- 关闭当前语音会话但不退出程序：说“再见”“拜拜”“休息吧”“结束对话”“退出”“结束”或“关闭”。程序会回到等待唤醒状态。

如需调整休眠、唤醒词、音频保存等行为，请修改 ``config.py`` 中对应的 ``AppConfig`` 配置；修改后需要重启程序。


交互流程
=====================
.. code-block:: text

   启动程序
   ↓
   等待唤醒词（IDLE）
   ↓ 识别到唤醒词
   播放提示音并播报“你好呀，我是派仔！很高兴见到你”
   ↓
   聆听本轮语音（READY）
  ↓
   语音识别 → 意图路由
   ├─ 聊天 / 问答 / 搜索：生成回答并播报
   ├─ 动作：规划、校验并执行机器人动作
   └─ 退出：回到等待唤醒状态
   ↓
   回答结束后继续聆听；约 15 秒无有效交互则自动休眠


当前默认配置下，一轮语音最长等待约 16 秒；过短语音或只有“嗯、啊、哦”等填充词会被忽略。

播报完成后可以直接继续说下一句；若需要中断播报，可再次说唤醒词，具体是否支持自由打断取决于配置，默认关闭自由打断。

常见问题与故障排查
============================

使用中遇到问题时，请先查阅以下常见问题和解决方案。

**Q1：我说了唤醒词，派仔没反应怎么办？**

**A：**\ 请依次检查：
* 确认麦克风已正确连接且未被静音

* 检查环境噪音是否过大（尝试走近到 1m 左右再说）

* API Key 已正确设置；

* 程序启动时使用了 --stt --tts 参数；

**Q2：我说了指令，但派仔理解成了别的东西？**

**A：**\ 语音识别在嘈杂环境下可能出现同音字错误（如将"挥手"识别为"回手"）。系统有内置的 ASR 纠错机制，大多数情况下能自动修正。如果持续识别不准，请检查环境噪音是否超标，或尝试换一种表达方式。

**Q3：我说了手势指令（如「挥挥手」），派仔回答但机器人没有动？**

**A：**\

* 动作指令未被正确识别；

* 机器人当前状态不允许执行该动作；

* 软件未连接真机；

* 动作被安全策略拦截。


**Q4：派仔说话时我能打断吗？**

**A：**\ 当前版本默认不支持语音打断，派仔在说话期间麦克风处于静音状态。但如果使用科大讯飞的麦克风，支持打断。.

**Q5：如何确认是否保存了录用**

**A：**\ 请检查 AUDIO_SAVE_ENABLED 的配置值：

未设置，或设置为 0、false 时，默认不持续保存录音；

设置为 1 或 true 后，录音将写入 audio/ 或指定目录。
